# meldlane-transcribe

[English version](README.md)

![Python](https://img.shields.io/badge/python-3.11%2B-blue) ![License](https://img.shields.io/badge/license-MIT-green) ![Platform](https://img.shields.io/badge/platform-Windows%20%7C%20macOS-lightgrey)

**Транскрибация встреч, звонков и голосовых сообщений**, которая запускается где угодно: превращает аудио/видео файл **или живую запись микрофона + системного звука** в структурированный JSON-транскрипт с таймлайном и спикерами. Работает на [faster-whisper](https://github.com/SYSTRAN/faster-whisper) (CTranslate2) — быстро на CPU, без torch, без внешнего ffmpeg для аудиоформатов.

Самостоятельный инструмент; также первый строительный блок тулчейна Meldlane (project management для AI-first команд).

Ключевые слова: транскрибация аудио, распознавание речи, speech-to-text, whisper на русском, транскрипт встречи, запись звонка, диаризация спикеров, Zoom/Google Meet/Telegram звонки в текст, offline распознавание речи, локальный whisper без интернета.

## Зачем

- **Без настройки**: `pip install`, запустил — готово. Ни ffmpeg, ни GPU, ни облачных API-ключей.
- **Спикеры бесплатно**: живая запись пишет микрофон и системный звук раздельными дорожками — разделение «я / собеседники» не требует вообще никакой ML-модели.
- **Работает на Windows без установки драйверов**: WASAPI loopback ловит системный звук с любого устройства вывода (колонки, наушники, Bluetooth) из коробки.
- **Структурированный вывод**: единый JSON-контракт (`meeting_id`, `segments[{start, end, speaker, text}]`) плюс обычный `.txt` для чтения глазами.

## Установка

```bash
pip install meldlane-transcribe        # или: uvx meldlane-transcribe --help
```

Python 3.11+. Windows и macOS.

## Использование

```bash
# транскрибировать файл (mp3, wav, m4a, ogg, aac, flac + видео mp4/mkv/webm/...)
mtranscribe file meeting.mp3

# живая запись (mic + системный звук, если найдено loopback-устройство), остановка из другого терминала
mtranscribe record
mtranscribe stop

# или запись фиксированной длительности
mtranscribe record --seconds 300

# диагностика: какие устройства найдены, какая стратегия захвата сработает
mtranscribe doctor
```

Результат: `outputs/<время>/transcript.json` (+ `transcript.txt` рядом — обычные строки `[mm:ss speaker] текст` для чтения).

```json
{
  "meeting_id": "a1b2c3",
  "lang": "ru",
  "duration_sec": 3612.4,
  "source": {"type": "live", "path": null},
  "created_at": "2026-07-02T18:00:00Z",
  "segments": [
    {"start": 0.0, "end": 4.2, "speaker": "me", "text": "..."}
  ]
}
```

## Спикеры без ML

Во время живой записи микрофон и системный звук пишутся **раздельными дорожками**, поэтому разделение по говорящим достаётся бесплатно: всё с микрофона — `me`, всё с loopback-устройства — `others`. Никакой модели диаризации, GPU, токенов.

**Ограничение:** так различаются только «я» и «все остальные» — если с той стороны звонка говорят несколько человек, все они попадут в `others` неразделённо, потому что у канала нет понятия отдельных голосов. Разделить их между собой можно только через голосовую диаризацию.

Для одноканальных файлов, или чтобы разбить `others` на отдельных спикеров, доступна полная диаризация (`spk_0`, `spk_1`, ...) как опциональный extra: `pip install meldlane-transcribe[diarize]` (pyannote, требует torch и бесплатный токен HuggingFace) — сознательно не в ядре, чтобы сохранить обещание установки без настройки.

## Захват системного звука

Каскад стратегий, побеждает первая доступная:

| ОС | Как | Настройка |
|----|-----|-----------|
| Windows | **WASAPI loopback** (через `pyaudiowpatch`) | не требуется — работает из коробки с любым устройством вывода |
| Windows (запасной вариант) | Стерео микшер / VB-Cable (автодетект) | включить «Стерео микшер» в настройках звука, или установить VB-Cable |
| macOS | BlackHole (автодетект) | `brew install blackhole-2ch` |

Нет loopback-устройства? `mtranscribe record` всё равно работает с одним микрофоном, а `mtranscribe doctor` подскажет, что делать и какая стратегия сейчас активна.

## Настройки (переменные окружения, все опциональны)

| Переменная | По умолчанию | Значение |
|-----|---------|---------|
| `MTRANSCRIBE_MODEL` | `base` | модель Whisper: tiny/base/small/medium/large |
| `MTRANSCRIBE_LANGUAGE` | авто | принудительный язык, напр. `ru` |
| `MTRANSCRIBE_MIC_DEVICE` | системный по умолчанию | подстрока имени микрофона |
| `MTRANSCRIBE_SYSTEM_DEVICE` | автодетект | подстрока имени loopback-устройства |
| `MTRANSCRIBE_OUTPUTS` | `./outputs` | где хранятся сессии |

Аудиодорожки удаляются после транскрибации, если не передан флаг `--keep-audio`.
